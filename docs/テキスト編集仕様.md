# テキスト編集機能 仕様書

## 概要

画像内のテキストを OCR で検出し、指定したテキストを別のテキストに置換する機能。ImgCraft のモードの一つとして提供。

### 対応フォーマット

| 種別 | 形式 |
|------|------|
| 入力 | PNG, JPEG, WebP, BMP |
| 出力 | PNG, JPEG, WebP（ユーザー選択、品質 95） |

### 処理の分離

- **検出**: 「テキストを検出」クリック時に OCR を 1 回実行
- **置換**: 「処理を実行」時に **再度 OCR を実行**（検出結果はキャッシュしない。bbox が必要なため再検出が必須）

---

## 1. テキスト検出（OCR）

### 1.1 使用エンジン

| 優先度 | エンジン | モデル | 備考 |
|--------|----------|--------|------|
| 1 | PaddleOCR | PP-OCRv5_mobile_det（検出）+ PP-OCRv5_mobile_rec（認識） | 日本語対応、高速 |
| 2 | EasyOCR | Reader(["ja", "en"]) | PaddleOCR 利用不可時のフォールバック |

### 1.2 前処理

- **拡大**: 最短辺が 800px 未満の画像は 800px まで拡大。トリミング画像（最短辺 < 400px）は 1200px まで拡大

### 1.3 後処理

- **ノイズ除外**: 1文字かつ信頼度が低い検出を削除（全体: 92%未満、トリミング: 80%未満）
- **隣接ボックス結合**: トリミング画像のみ有効（「AmazonP」+「ay」→「AmazonPay」のように補完）

### 1.4 API

- **エンドポイント**: `POST /detect-text`
- **リクエスト**: `multipart/form-data`、`file` に画像
- **レスポンス**: `{"texts": [{"text": str, "confidence": float}, ...], "engine": "paddleocr"|"easyocr"}`

## 2. テキスト置換

### 2.1 マッチング条件

1. **完全一致（部分一致）**: 置換元テキストが OCR 結果のいずれかに**含まれる**（大文字小文字無視）
2. **類似度マッチ**: 完全一致がなければ、類似度 75% 以上の最も近い結果を採用（編集ミス・OCR 差を吸収）

### 2.2 置換処理の流れ

1. 画像を読み込み
2. OCR でテキストを検出
3. 置換元に一致する領域を特定
4. 各一致領域について:
   - 元テキストの**文字色**を領域から推定（背景と最も異なる色）
   - 領域の**背景色**を外側から推定
   - テキスト領域を**背景色で塗りつぶし**（単色、ぼやけ防止）
   - 新しいテキストを描画

### 2.3 描画仕様

| 項目 | 仕様 |
|------|------|
| フォント | ヒラギノ角ゴ/丸ゴ、Arial Bold、Impact、Noto Sans CJK 等（環境に存在するものから選択） |
| フォントサイズ | bbox の 70% 以内に収める（下のテキストを覆わないよう厳しめに） |
| 配置 | 中央寄せ |
| 文字色 | 元テキスト領域から推定（背景と最も異なる色） |
| 縁取り | 白・明るい文字: 縁なし / 暗い文字: 1〜3px（フォントサイズの約 6%） |
| 縁の色 | 白系: 同色 / 暗い文字: 明るく / 金・オレンジ系: 暗い茶色 |

### 2.4 背景塗りつぶし

- **方式**: 単色塗り（`cv2.fillPoly`）
- **色**: テキスト領域の外側（パディング 15%）から中央値を取得
- **理由**: inpaint によるぼやけを避けるため

## 3. UI 仕様

### 3.1 操作フロー

1. 画像を選択
2. （オプション）編集範囲をドラッグで指定すると OCR 精度が向上
3. 「テキストを検出」をクリック
4. 検出結果がチップとして表示（件数と使用エンジンも表示）
5. チップを**クリック** → 置換元入力欄にコピー
6. チップの **✎** をクリック → インラインで編集（Enter で確定、Escape でキャンセル）
7. 置換元・置換先を入力
8. 「処理を実行」で置換（範囲指定時はその範囲のみ処理し、結果を貼り戻す）

### 3.2 検出結果チップ

- テキスト + 信頼度（%）を表示
- クリック: 置換元にコピー
- ✎: 編集モード（編集後のテキストは置換に使用可能、類似度マッチで対応）

### 3.3 バリデーション

- 置換元・置換先の両方が必須
- 画像未選択時は処理不可

## 4. エラーメッセージ

| 状況 | メッセージ |
|------|------------|
| OCR 未インストール | 「OCR が必要です: pip install paddleocr または pip install easyocr」 |
| 検出失敗（API） | 「検出に失敗しました」（サーバーエラー時は EasyOCR のインストール確認を促す） |
| 検出結果なし（UI） | 「テキストが検出されませんでした」 |
| 置換マッチなし | 「『{置換元}』が見つかりませんでした。OCRで検出されたテキストを確認してください。」 |
| 画像未選択 | 「画像を選択してください」 |
| 動画選択時 | 「動画は対応していません。画像を選択してください」 |
| 置換元・置換先未入力 | 「置換元・置換先のテキストを両方入力してください。」 |

## 5. パフォーマンス

- **初回起動**: PaddleOCR は初回にモデル（PP-OCRv5_mobile_det / rec）をダウンロードするため、数十秒〜数分かかることがある
- **検出・置換の二重実行**: 置換実行時に OCR を再実行するため、検出と置換で合計 2 回 OCR が走る
- **推奨**: 大容量画像は事前にリサイズしてからテキスト編集を行うと高速化できる

## 6. 制限・注意事項

- **動画**: 非対応
- **フォント**: システムフォントのみ。元画像のデザインフォントは再現不可
- **グラデーション背景**: 単色塗りになるため、グラデーション部分はやや不自然になりうる
- **複数一致**: 同じ置換元に複数マッチする場合、すべて置換される

---

## 7. 依存関係

- **必須**: `paddleocr` または `easyocr` のいずれか
- **推奨**: `paddleocr`（日本語精度・速度のため）
- **その他**: `opencv-python`, `numpy`, `Pillow` (8.0+ で stroke 対応)
