# CTA アニメーション 仕様書

## 概要

画像にアニメーション効果を付与し、GIF または埋め込み用ソースコード（HTML+CSS/JS）を生成する機能。LP（ランディングページ）の CTA ボタンやバナー画像に動きを付け、クリック率向上を目的とする。

### 対応フォーマット

| 種別 | 形式 |
|------|------|
| 入力 | PNG, JPEG, WebP, BMP（画像のみ） |
| 出力 | GIF / ソースコード（HTML+CSS/JS）/ 両方（ZIP） |

### 処理の流れ

1. ユーザーが画像を選択し、アニメーション効果と出力形式を指定
2. **GIF 出力時**: フレーム単位で画像にエフェクトを適用し、GIF を生成
3. **ソースコード出力時**: エフェクトに応じた HTML+CSS（必要に応じて JS）を生成
4. **両方出力時**: GIF とソースコードを ZIP でまとめて返却

---

## 1. アニメーション効果

全 21 種類のエフェクトを提供。CSS のみで実現するものと、JavaScript を必要とするものに分類される。

### 1.1 CSS のみ（16 種類）

| エフェクト ID | ラベル | 説明 |
|---------------|--------|------|
| pulse | パルス | 拡大縮小＋明るさ変化 |
| glow | グロー | 外側に光が広がる |
| bounce | バウンス | 上下に弾む |
| shake | シェイク | 左右に揺れる |
| float | フロート | ゆっくり上下に浮く |
| wiggle | ウィグル | 小刻みに揺れる |
| fade | フェード | 透明度が脈打つ |
| rotate | ローテート | 軽く回転 |
| swing | スイング | 振り子のように揺れる |
| heartbeat | ハートビート | 心拍のような動き |
| rubber | ラバー | ゴムのように伸び縮み |
| breathe | ブリーズ | ゆっくり膨らむ・縮む |
| attention | アテンション | 一瞬大きく動いて注目を集める |
| shine | シャイン | 光が走る（CSS 疑似要素） |
| ripple | リップル | クリックで波紋が広がる |
| tilt | チルト | 3D で傾いて浮く |

### 1.2 JavaScript 使用（5 種類）

| エフェクト ID | ラベル | 説明 | GIF 出力 |
|---------------|--------|------|----------|
| magnetic | マグネティック | カーソルに吸い寄せられる | 不可 |
| parallax | パララックス | マウスで 3D 傾き | 不可 |
| spotlight | スポットライト | カーソルに光が追従 | 不可 |
| particle | パーティクル | クリックで粒子が飛ぶ | パルスで代用 |
| cursor_glow | カーソルグロー | カーソルに光が追従 | 不可 |

**GIF 出力不可の理由**: マウス位置・クリックに依存するため、静的アニメーションである GIF では再現できない。これらのエフェクト選択時は UI 上で「ソースコード」のみ選択可能。

## 2. 出力形式

### 2.1 GIF でダウンロード

- 1 周期あたり約 1.5 秒、20fps でフレーム生成
- ループ再生（無限）
- 単一 GIF ファイルとしてダウンロード

### 2.2 ソースコード

- 埋め込み用の HTML スニペットを生成（`<!DOCTYPE>` や `<html>` タグは含まない）
- LP の任意の位置にそのまま貼り付け可能
- 画像パスは `YOUR_IMAGE.png` のプレースホルダで出力し、ユーザーが任意のパスに変更する前提

### 2.3 両方

- GIF とソースコードを ZIP にまとめて返却
- ZIP 内のファイル名: `{stem}_cta.html`, `YOUR_IMAGE.gif`
- 解凍後、そのまま HTML が動作する

---

## 3. 画像パス

### 3.1 プレースホルダ

| 出力形式 | プレースホルダ | ZIP 内の画像ファイル名 |
|----------|----------------|------------------------|
| ソースコードのみ | YOUR_IMAGE.png | YOUR_IMAGE.png |
| 両方 | YOUR_IMAGE.gif | YOUR_IMAGE.gif |

### 3.2 ユーザーへの案内

生成コードの先頭にコメントを付与:

```html
<!-- 画像パス（YOUR_IMAGE.png）を任意のパスに変更してください -->
```

LP に貼り付ける際は、`YOUR_IMAGE.png` を実際の画像パス（相対パス・絶対パス・URL）に置き換える。

## 4. 生成コードの形式

### 4.1 CSS エフェクトの基本構造

```html
<style>
.cta-wrap { display: inline-block; animation: ...; }
@keyframes cta-xxx { ... }
.cta-wrap img { display: block; max-width: 100%; height: auto; }
</style>
<div class="cta-wrap">
  <img src="YOUR_IMAGE.png" alt="CTA">
</div>
```

### 4.2 JS エフェクトの基本構造

```html
<style>...</style>
<div class="cta-wrap" data-cta-xxx>
  <img src="YOUR_IMAGE.png" alt="CTA">
</div>
<script>
(function() {
  document.querySelectorAll('[data-cta-xxx]').forEach(function(wrap) {
    // イベント処理
  });
})();
</script>
```

- 複数埋め込みを想定し、`data-cta-*` 属性と `querySelectorAll` で全要素にイベントを付与
- IIFE でスコープを隔離し、他スクリプトとの衝突を防止

## 5. API・処理フロー

### 5.1 エンドポイント

- **POST**: `/process`（mode=cta）

### 5.2 リクエストパラメータ

| パラメータ | 説明 | 値 |
|------------|------|-----|
| mode | 処理モード | cta |
| cta_effect | エフェクト ID | 上記のいずれか |
| cta_output | 出力形式 | gif / code / both |
| file | 画像ファイル | multipart |

### 5.3 バリデーション

- `cta_effect` が `CTA_EFFECTS_NO_GIF` に含まれる場合、`cta_output` が `gif` または `both` だと 400 エラー
- エラーメッセージ: 「このエフェクトはカーソル操作に依存するため、GIF出力はできません。ソースコードを選択してください。」

### 5.4 レスポンス

- **GIF のみ**: `application/octet-stream` または `image/gif`、単一 GIF
- **ソースコードのみ**: ZIP（`{stem}_cta.html` + `YOUR_IMAGE.png`）
- **両方**: ZIP（`{stem}_cta.html` + `YOUR_IMAGE.gif`）

## 6. UI 仕様

### 6.1 出力形式の表示切り替え

- カーソル依存エフェクト（magnetic, parallax, spotlight, cursor_glow）選択時:
  - 「GIF でダウンロード」「両方」を非表示
  - 「ソースコード」のみ表示し、自動選択
  - ヒント: 「このエフェクトはカーソル操作に依存するため、ソースコードのみ出力できます。」

### 6.2 プレビュー

- 画像選択時、選択中のエフェクトをリアルタイムでプレビュー表示
- リップル: クリックで波紋をシミュレート
- JS エフェクト: パルスアニメーションで代用表示（プレビューではマウス操作を再現できないため）

### 6.3 ソースコードプレビュー

- ソースコード出力時、生成結果をプレビューエリアに表示
- クリックでコピーボタンにより、コードをクリップボードにコピー可能
- カーソル: 画像時は `zoom-in`、コード時は `text`

---

## 7. 依存関係

- **PIL (Pillow)**: 画像読み込み・フレーム生成
- **NumPy**: 画像処理

GIF 生成には外部ライブラリの追加は不要。ソースコード生成は HTML 文字列の組み立てのみ。
